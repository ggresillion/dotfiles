# NixOS Flake — Agent Guide

## Quick rebuild
```bash
sudo nixos-rebuild switch --flake /etc/nixos
```
Alias defined in `home/common/shell.nix` as `nswitch`. Also `nedit` for editing config.

## Structure
- `flake.nix` — single host `guillaume-desktop` (x86_64-linux). Only one NixOS config.
- `hosts/desktop/` — system-level modules (hardware, gaming, disko, plex, jellyfin, vr, rgb, storage, greeter, appimage).
- `home/` — home-manager user profiles. `home/common/` is shared between users.
- Users: `guillaume` (primary), `guillaume_thetreep` (adds slack + 1password).
- `config/Makefile` under `home/common/config/` — GNU Stow targets for dotfiles (legacy, not used by home-manager activation).

## Style / conventions
- Use `{ pkgs, inputs, ... }:` function args — not `self` unless needed.
- Dotfiles in `home/common/config/<app>/.config/<app>/` — home-manager copies via `xdg.configFile`.
- No `opencode.json`, `.cursorrules`, or prior instruction files exist.
- `home-manager.useGlobalPkgs = true; useUserPackages = true;` — HM shares NixOS pkgs.

## Gotchas
- Millennium Steam overlay is **disabled** (issue #551). Uncomment when fixed.
- `nix-index-database` is a NixOS module, not HM.
- `noctalia.kdl` is gitignored (generated by Noctalia activation script).
- WezTerm color schemes are gitignored.
- Neovim uses `neovim-nightly-overlay` + extra packages (nixd, nixfmt, tree-sitter).
- `flake.nix` passes `inputs` as `specialArgs` — available in all modules.
- Binary caches configured in `configuration.nix`: nix-community, neovim-nightly-overlay, nix-gaming.

## Verification
- `nix flake check` — validates flake schema.
- `nixos-rebuild dry-build` — dry run before switch.
- No test harness, no CI pipeline.
